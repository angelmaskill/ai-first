# Wiki Generator

Generate and update project wiki pages from the project's accumulated knowledge,
architecture decisions, standards, and code structure. The wiki is the
human-readable index of everything the project knows about itself.

## When to Use This Skill

- After stage transitions (especially architecture → scaffold, build → qa)
- After major feature completion
- When /wiki is invoked
- When knowledge-sync-agent flags stale wiki content

## Workflow

### Phase 1: Gather Sources

Collect all available information:

```bash
echo "=== Knowledge items ==="
find .ai-first/knowledge -name "*.md" 2>/dev/null | head -20

echo "=== Standards ==="
find .ai-first/standards -name "*.md" 2>/dev/null

echo "=== Artifacts ==="
ls .ai-first/artifacts/ 2>/dev/null

echo "=== Architecture ==="
test -f .ai-first/artifacts/architecture.md && echo "present" || echo "absent"

echo "=== Requirements ==="
test -f .ai-first/artifacts/requirements.md && echo "present" || echo "absent"

echo "=== Recent reviews ==="
ls -t .ai-first/reviews/ 2>/dev/null | head -5

echo "=== Recent reports ==="
ls -t .ai-first/reports/ 2>/dev/null | head -5
```

### Phase 2: Generate Wiki Pages

For each detected category, generate (or update) a wiki page:

#### 2.1 Project Overview → `wiki/overview.md`
- Project name, description, current stage
- Tech stack summary
- Team mode and code domains

#### 2.2 Architecture → `wiki/architecture.md`
- Module map from architecture.md
- Key ADRs with decisions and rationale
- Scaffold layout

#### 2.3 API Reference → `wiki/api-reference.md`
- Extract from knowledge items with category=api-contract
- Endpoint signatures, request/response types

#### 2.4 Data Model → `wiki/data-model.md`
- Extract from knowledge items with category=data-model
- Key entities, relationships, schemas

#### 2.5 Standards → `wiki/standards.md`
- Aggregate from .ai-first/standards/
- Coding conventions, review checklist, security policies

#### 2.6 Development Guide → `wiki/dev-guide.md`
- Setup instructions
- Build and test commands
- Common workflows

#### 2.7 Changelog → `wiki/changelog.md`
- Stage transitions completed
- Major features delivered
- Breaking changes

### Phase 3: Wiki Index

Generate `wiki/index.md` — a navigable table of contents:

```markdown
# Wiki: {project name}

**Last updated**: {timestamp}
**Current stage**: {stage}

## Pages
- [Overview](overview.md)
- [Architecture](architecture.md)
- [API Reference](api-reference.md)
- [Data Model](data-model.md)
- [Standards](standards.md)
- [Development Guide](dev-guide.md)
- [Changelog](changelog.md)
```

### Phase 4: Cross-Reference Check

Verify wiki pages don't contradict each other:
- API ref endpoints should match architecture module contracts
- Data model entities should match architecture module map
- Standards should be referenced in dev-guide

Flag any inconsistencies found.

## Output Format

Each wiki page follows this template:

```markdown
# {Page Title}
**Last updated**: {timestamp}
**Source**: {where this information came from}

{content}

---
*Generated by wiki-generator. Update sources in .ai-first/, then re-run to refresh.*
```

## Quality Standards
- Every wiki page MUST cite its source (which artifact/report/knowledge item)
- Wiki pages MUST NOT duplicate — link between pages instead
- Generated content MUST be verifiable against source
- If a source is absent (e.g., no architecture.md), write "Not yet available" with a note on how to create it
