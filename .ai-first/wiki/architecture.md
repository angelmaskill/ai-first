## Module Design

The AI-First scaffold is organized into three layers:

### Control Layer (`.ai-first/`)
- `routing.yml` — Authoritative agent routing table (single source of truth for dispatch)
- `project.yml` — Project identity, stage, and domain configuration
- `state/` — Stage tracking via symlink (`current → stage-XX-{name}`)
- `locks/` — Soft/hard lock enforcement during execution stages

### Agent Layer (`.claude/agents/`)
15 specialized agents, each defined as a Markdown file with constraints:
- **opus-tier**: intake-agent, planner-agent, architect-agent, team-lead-agent
- **sonnet-tier**: builder-agent, reviewer-agent, security-reviewer-agent, release-agent, repo-scanner-agent, stage-assessor-agent, knowledge-sync-agent, smoke-case-agent
- **haiku-tier**: state-updater-agent, skill-recommend-agent, marketplace-skill-agent

### Skill Layer (`.claude/skills/`)
7 skills following `SKILL.md` convention: security-scan, bug-scan, optimization-scanner, wiki-generator, prd-generator, code-scaffold, test-generator

## Agent Routing Protocol

Dispatch follows a deterministic protocol:
1. Slash commands → direct lookup in `routing.yml` → immediate dispatch
2. Free-text requests → intent classification → keyword match against routes → confirmation gate
3. Complex tasks (>0.6 complexity) → subagent-dispatcher.ts topological sort

## Quality Gate Architecture

9 review gates per task, run in two parallel lanes:
- **Reviewer lane**: logic, architecture, architecture_risk, docs, testing, consistency, collaboration
- **Security lane**: security (OWASP Top 10, secrets, dependencies)

Post-build chain: bug-scan + security-scan → knowledge-sync → reviewer + security-reviewer → state-update

## Stage Transition Model

```
idea → discovery → spec → architecture → scaffold → build → qa → release → operate → evolve
                ↑        ↑              ↑          ↑       ↑  ↓       ↑         ↓
                └────────┴──────────────┴──────────┴───────┘  └───────┘         └────→ discovery
```

Each transition validates an exit checklist before advancing.
