### STANDARD-001: API Contract Consistency
API contracts must be versioned and typed. Frontend and backend must agree on request/response shapes. Changes to contracts require ChangeScope declarations that cover both sides.

### STANDARD-002: Agent Domain Boundaries
Each agent has a defined domain — reviewer MUST NOT override security-reviewer, builder MUST NOT review. Cross-domain work dispatches multiple agents.

### STANDARD-006: Deterministic Agent Routing Protocol
Agent dispatch uses `routing.yml` as the single source of truth. Agent `.md` descriptions are for internal context only. Confirmation gate required for free-text requests.

### STANDARD-004: Code Review Process
7 review gates. Parallel execution of reviewer + security-reviewer. Max 3 fix iterations per task.

### STANDARD-005: Knowledge Synchronization
Knowledge-sync-agent runs after every build task. Sync events follow lifecycle: `suggested → pending → confirmed → resolved`. Knowledge items have reviewDate-based expiry tracking.

### STANDARD-007: Test Quality Standards
Test pyramid: unit > integration > e2e > smoke. Test:source ratio targets defined. Test-to-production boundary rules enforced.
