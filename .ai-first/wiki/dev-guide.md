## Pre-Commit Hook

Runs automatically on `git commit`. Checks 6 things:
1. **Secrets detection** — API keys, tokens, passwords, private keys
2. **Env file blocking** — `.env`, `.pem`, `.key` files blocked
3. **Empty catch detection** — blocks `catch (e) {}`
4. **Console statements** — errors if >20 per file
5. **TypeScript typecheck** — `tsc --noEmit`
6. **Tests** — `vitest run --changed`

## CI Pipeline

Triggered on push/PR to master/main. Four jobs:
1. `typecheck` — Full TypeScript check
2. `test` — Full vitest suite
3. `security-scan` — Secret detection + gitignore audit + npm audit
4. `bug-scan` — Console limit + empty catch check

## Common Workflows

### Fix a bug
1. Create task: `/task "fix: description"`
2. Implement changes (delegated to builder-agent)
3. Complete: `/complete [task-id]`
4. Pre-commit hook runs automatically on commit

### Add a feature
1. Plan: `/decide "feature decision"`
2. Task: `/task "feat: description"`
3. Build → /complete
4. Review gates validate automatically

### Check project health
```
/health
```
Shows: current stage, test coverage, review status, knowledge sync, risk summary.

### Get guidance on next steps
```
/guide
```
Shows: recommended next action with rationale, alternative actions, risks.
